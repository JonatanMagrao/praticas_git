Comandos básicos do Git na linha de comando

  • git init

    Cria um novo repositório .git

  • git config --global user.name "nome-autor-versão"

    Comando para criar um autor para a criação daquele commig. --global é para a criação em todos os repositórios da máquina. Quando não declarado, o usuário é criado apenas naquele repositório local.

  • git config --global user.email "email-autor-versão"

    Segue a mesma premissa para o do nome

  • git clean -f

    Comando que limpa as alterações de tudo o que foi feito em arquivos que não foram trackeados (adicionados)

  • git restore nome_arquivo

    Comando que limpa as alterações de arquivos que já foram adicionados (criados commits)

  • git restore --staged nome_arquivo

    Comando que limpa as alterações de arquivos que já foram adicionados (criados commits)

  • git status

    Comando para verificar os arquivos e pastas que foram modificados e não salvos/commitados do repositório

  • git add nome-arquivo ou . ou * para salvar todos

    Comando para salvar (criar uma versão) os arquivos e pastas que foram modificados

  • git commit -m "breve comentário sobre a nova versão"

    Comenta (commit) o que foi alterado nesta nova versão. Como boa prática, deve-se fazer um commit sempre que o uma nova versão for criada.

  • git tag "nome-tag"

    Cria uma tag para aquela versão (commit)

  • git log

    Mostra todas as versões criadas, junto com os commits e a branch atual

  • git branch "nome-branch"

    Mostra todas as branchs criadas e em qual você está atualmente. Adicionando o nome depois, cria uma nova branch

  • git branch -d nome-branch

    Deleta a branch nomeada

  • git checkout nome-tag, numero, nome-branch

    Comando para navegar entre as commits e branch. Adicionando o nome de um arquivo ou pasta, apenas aquele arquivo ou pasta navega entre os commits ou branchs

  • git rm nome-arquivo

    Deleta o arquivo nomeado naquela commit

  • git diff

    Comando para mostrar as diferenças entre as versões. Deve ser feito depois de ter feito alguma alteração e antes de criar um novo commit.

  • git rm -rf .git

    Apaga a pasta .git do repositório. Deve-se ter muito cuidado ao realizar essa tarefa, pois apaga tudo o que nela continha (commits, versões, etc)

  • git merge

    Junta duas ramificações em uma só. Ex: estou no branch master, então uso git merge nome-branch

  • git clone endereço-web

    Clona um repositório na web para um armazenamento local

  • git remote -v

    Mostra o endereço web daquele repositório (quando estiver disponível na web)

  • git push nome-local + nome-branch

    Atualiza repositório na web. Ex: git push origin master

  • git pull

    Atualiza repositório local com o que está disponível na web
